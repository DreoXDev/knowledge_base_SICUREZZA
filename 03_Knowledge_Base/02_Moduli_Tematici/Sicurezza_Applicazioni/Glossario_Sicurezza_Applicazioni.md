# Glossario Sicurezza Applicazioni

## Security policy

Insieme di regole che determina come un sistema o un'organizzazione protegge risorse critiche e sensibili.

## Weakness

Tipo di errore nel progetto, nell'implementazione o nell'operazione di un sistema software che, in condizioni appropriate, puo' portare all'introduzione di vulnerabilita'.

## Vulnerabilita'

Occorrenza concreta di una o piu' weakness in un sistema software, sfruttabile da una terza parte per violare la security policy.

## Exploit

Tecnica concreta che permette di sfruttare una vulnerabilita' per violare la security policy.

## Attacco e attack pattern

Un attacco e' un'azione intenzionale volta a violare la security policy. Un attack pattern e' uno schema generale di attacco, riusabile per descrivere come una certa classe di weakness puo' essere sfruttata.

## Attacker

Entita' che effettua un attacco. Puo' essere interna o esterna al perimetro di sicurezza.

## Countermeasure

Azione o tecnica che elimina, previene, mitiga o individua una vulnerabilita', una minaccia o un attacco.

## Caratteristiche degli attacchi

| Dimensione | Tipi | Significato |
|---|---|---|
| Tipologia | attivo / passivo | altera risorse e operazioni, oppure acquisisce informazioni |
| Point of initiation | interno / esterno | parte da dentro o fuori il perimetro di sicurezza |
| Method of delivery | diretto / indiretto | colpisce il target direttamente o tramite terzi |

## Da non confondere

- Weakness: categoria di difetto.
- Vulnerabilita': occorrenza concreta nel sistema.
- Exploit: tecnica concreta.
- Attack pattern: schema generale.
- Countermeasure: prevenzione, mitigazione, detection o correzione.
