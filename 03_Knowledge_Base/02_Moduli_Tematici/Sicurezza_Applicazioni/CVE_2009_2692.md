# CVE-2009-2692

CVE-2009-2692 e' una vulnerabilita' del kernel Linux legata a null pointer dereference in codice socket.

## Punto vulnerabile

La funzione `sock_sendpage` usava un puntatore a funzione nella struttura socket senza verificare correttamente che fosse non NULL. Il confronto con `sock_splice_read` nelle slide evidenzia la differenza: la funzione corretta controlla il puntatore prima dell'uso.

## Trigger

L'attacco puo' essere innescato tramite `sendfile`, arrivando al percorso vulnerabile e causando l'uso di `sock->ops->sendpage` quando il puntatore non e' valido.

## Exploit

L'attaccante mappa la pagina NULL, prepara codice o strutture controllate e induce il kernel a dereferenziare il puntatore. Poiche' il codice gira in kernel mode, l'esito puo' essere una shell root.

## Lezione principale

Una null pointer dereference di puntatori a funzione nel kernel non e' solo un crash: puo' diventare privilege escalation se l'attaccante controlla l'indirizzo dereferenziato.
