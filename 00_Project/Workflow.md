# Workflow

## Processo operativo

1. Inserire un nuovo asset nella sottocartella corretta di `01_Raw_Assets/`.
2. Registrare l'asset in `00_Project/Source_Register.md`.
3. Analizzare l'asset esternamente o tramite uno strumento AI.
4. Produrre un piano di aggiornamento per Codex.
5. Creare o aggiornare la source note in `02_Source_Notes/`.
6. Integrare i concetti in `03_Knowledge_Base/`.
7. Aggiornare `04_Exam_Prep/` con domande, risposte, flashcard o schemi.
8. Controllare duplicazioni, coerenza dei link e tracciabilita' delle fonti.
9. Aggiornare lo stato della fonte in `Source_Register.md`.

## Regole per gli asset

Gli asset originali non devono essere modificati. Se un file deve essere escluso dal flusso principale, va spostato in `99_Archive/` solo su decisione esplicita.

Ogni asset deve avere un ID fonte progressivo nel formato `SRC-001`, `SRC-002`, `SRC-003`.

## Regole per l'integrazione

Prima di creare nuove note, cercare contenuti simili in:

- `03_Knowledge_Base/`
- `04_Exam_Prep/Risposte_Da_Esame/`
- `04_Exam_Prep/Flashcards/`

Se una nota correlata esiste gia', aggiornarla e aggiungere la nuova fonte nella sezione `Fonti`.

## Controllo qualita'

Prima di concludere ogni aggiornamento, verificare:

- nessuna duplicazione evidente;
- indice aggiornato;
- link interni utili e coerenti;
- fonti tracciate;
- risposte d'esame coerenti con le note;
- concetti ambigui segnalati.
